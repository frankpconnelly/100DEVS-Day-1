I. Software Engineering as a career

  • High-paid
  • Happiest
  • Don't need a degree
  • Very difficult!

III. What is Software Engineering?
  • Everyone has their own defintion, but it's a professional who writes code and understands not only how to write code, but also the tooling and instructions that a computer can understand.
  • They might write web applications that run on a browser, apps on a phone, something that appears on your friedge, your coffee maker...
  • You do not need a 4 year degree. Write code, conduct yourself as a professional, know and understand the systems behind the code you're writing... you are a software engineer.
  • Software engineers have specialties. Our specialty will be a full stack web development specialty.
  • You could take these skills and work on mobile apps. Robots, etc. The fundamentals will be the same, but the languages and tooling may be different in other specialties.
IV. Why Web Development?
  • One language. JavaScript.
  • Full stack JavaScript is in high demand.
  • Web Development is NOT taught in four year universities.
  • This gives us an edge; if we can learn the practical skills needed, we can BACKFILL all the things we'd have learned in the four year program.
  • As I progress, I can fill in my data structures, my algorithms, all the things that a four year CS degree would teach.
V. How do I do this for free?
  • Learn the basics
  • Go through one full stack tutorial (this one)
  • Build a passion project that you sink hours into
  • Then, interview over and over and over!
VI. How can I be successful?
  1. Manage your frustration
    • You will fail. A lot. Get over it.
  2. Consistency
    • One hour a day. That's it. Be realistic. Pick a time that's reasonable. Talk with family.
  3. Take care of yourself
    • Learning to code is a marathon, not a sprint. Don't start hard and burn out; way too many people do that.

  • Learn something. By day 3, there's a 40% chance we don't remember it; however, reviewing it each day mitigates the chances of forgetting it.

1.3 What is the Internet?

I. The web is a wire. It connects a client-side device to a server. Both are computers.

  1. Client-side device can be anything; a phone, a computer, a scanner...
  2. Server is another computer across the wire. It has a very appropriate name; it serves.
    • Server serves up some information. That information finds its way to a client-side device. The info comes into the device, and we can consume or interact with that information.

1.4 HTML5? What is it?

I. 5th iteration of HTML.

  1. New features: video (no flash, no plugins; HTML5 just runs video natively.), audio (Soundcloud), canvas ("io" games), geolocation (allow/block location? Natively get location), websockets (Chat Roulette/Omegle)

The Golden Rule: Separation of Concerns

  1. Our HTML should ONLY be our content/structure.
  2. Our CSS should ONLY be our style.
  3. Our JS should ONLY be our behavior/interaction.
Simply put, if we work with a team (or come back to an old piece of code), know that if I wanted to add content, I'd go to HTML, style CSS, and interaction JS.
